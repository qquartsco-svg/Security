# MPK 보안 모델 설명

이 문서는 `MemoryPhase_Kernel`이 제안하는 보안 개념을 기존 로그인 방식과 비교해 설명한다.

## 1. 핵심 정의

MPK는 단순 로그인 패키지가 아니다.

- 일반 로그인은 “들어오게 할 것인가”를 묻는다.
- MPK는 “누가 이 정보를 자기 정보로 해석할 수 있는가”를 묻는다.

즉 이 패키지는 **기억 기반 접근 커널**이다.

## 2. 물과 얼음 비유

- `liquid`
  - 정보는 존재하지만 바로 개인 의미를 드러내지 않음
  - 마스킹, 요약, 비해석 상태

- `semi_frozen`
  - 일부 문맥만 정렬
  - 운영 정보 수준까지만 열림

- `frozen`
  - 사용자가 맞다고 충분히 판단될 때
  - 개인/민감 정보까지 의미 있게 정렬

## 3. 기억 기반 보안이란 무엇인가

MPK가 보는 “기억”은 raw 녹음/사진 더미가 아니다.

- channel weights
- trust history
- behavior tendencies
- long-term matching structure

즉 AI가 “이 사람이 누구인가”를 장기적으로 기억하고, 현재 신호와 비교하는 구조다.

## 4. 현재 패키지의 범위

현재 MPK가 제공하는 것:

- 기억 기반 동일성 점수
- 공명/동역학
- 붕괴 기반 접근 판단
- 위상 전이
- 민감도 티어 허용
- Vault materialize 스캐폴딩
- Session observer loop 스캐폴딩

현재 제공하지 않는 것:

- 실제 암호화 저장소
- 완전한 생체 인식 모델
- 완전한 장기 학습 시스템
- 완전한 감사 체인

## 5. 다른 방식과의 차이

### 비밀번호 기반

- 장점: 단순
- 한계: 유출되면 끝, 세션 중 재확인 약함

### 생체 인증 기반

- 장점: 강한 단일 채널
- 한계: 한 채널 오류나 위조에 민감

### 행동 기반 인증

- 장점: 세션 중 재확인 가능
- 한계: 구조화가 약하면 규칙 모음에 머무름

### MPK

- 장점:
  - 장기 기억
  - 다중 채널
  - 연속 인증
  - 위상 기반 정보 정렬
- 한계:
  - 설명과 설계가 복잡함
  - 실제 배포에는 trust anchor와 저장소 계층이 필요함

## 6. 왜 운영 시스템과 분리해야 하나

Atom / Athena / Pharaoh 같은 운영 코어는 결과를 만든다.

MPK는:

- 그 결과를 누가 읽을지
- 어떤 민감도까지 열릴지
- 언제 다시 닫힐지

를 결정한다.

즉 운영 판단과 접근 해석 권한을 분리하는 것이 핵심이다.

## 7. 확장 원칙

1. raw보다 feature 중심 저장
2. trust anchor 유지
3. phase 결정과 실제 복호 분리
4. session observer 강화
5. audit chain 별도 확장
6. 기억은 누적되되 forgetting/drift도 같이 넣기

## 한 줄 결론

**MPK는 “AI가 기억한 사용자에게만 정보가 얼음처럼 정렬되는가”를 결정하는 기억 기반 접근 보안 커널이다.**
